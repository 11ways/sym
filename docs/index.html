sym

A user-friendly symbolic link manager for ~/.local/bin

Installation

# Install using Homebrew (recommended) $ brew install 11ways/sym/sym # Verify installation $ sym --version sym version 1.0.1 # View documentation $ man sym
# One-line installation $ curl -fsSL https://raw.githubusercontent.com/11ways/sym/main/install.sh | bash # Or safer (inspect first) $ curl -fsSL https://raw.githubusercontent.com/11ways/sym/main/install.sh -o install.sh $ less install.sh $ bash install.sh # Or use Homebrew on Linux $ brew install 11ways/sym/sym
# Clone and install with Makefile $ git clone https://github.com/11ways/sym.git $ cd sym $ make install-local # Or quick setup $ chmod +x sym.sh $ ./sym.sh sym $(pwd)/sym.sh

Features

🔗

Easy Link Management

Create, list, inspect, and remove symbolic links with simple commands. Intuitive interface that just works.

🛡️

Safety First

Input validation, confirmation prompts, and dry-run mode ensure you never make mistakes with your links.

🔍

Verify & Fix

Automatically detect broken links and fix them with a single command. Keep your environment clean.

🎨

Beautiful Output

Color-coded interface with status indicators. Clear, aligned formatting makes everything easy to read.

📊

Multiple Formats

Export to text, JSON, or CSV. Perfect for automation, scripting, and integration with other tools.

📚

Full Documentation

Comprehensive man page, detailed README, examples, and contributing guidelines. Everything you need.

🚀

Cross-Platform

Works seamlessly on both macOS and Linux. Handles platform differences automatically.

⚡

Fast & Lightweight

Pure bash with no external dependencies. Instant startup, minimal footprint.

Quick Examples

Create a Symlink

$ sym sublime /Applications/Sublime\ Text.app/Contents/.../subl
✓ Successfully created symbolic link:
  → From: /Applications/Sublime Text.app/.../subl
  → To:   ~/.local/bin/sublime

List All Links

$ sym ls

Existing symbolic links in '~/.local/bin':

           sublime → /Applications/Sublime Text.app/.../subl
              code → /Applications/Visual Studio Code.app/.../code
            docker → /usr/local/bin/docker

Show Link Info

$ sym info sublime

  → From:    ~/.local/bin/sublime
  → To:      /Applications/Sublime Text.app/.../subl
  → Created: 2025-10-31 14:23:45
  → Size:    2.4MB
  → Type:    File

Verify Links

$ sym verify

Verifying symbolic links...

  ✓ sublime → /Applications/Sublime Text.app/.../subl
  ✓ code → /Applications/Visual Studio Code.app/.../code
  ✗ oldtool → ~/bin/removed-tool (broken)

✓ All 3 symbolic links verified!

Export to JSON

$ sym ls --format=json
[
  {
    "name": "sublime",
    "target": "/Applications/Sublime Text.app/.../subl",
    "status": "ok",
    "size": "2.4MB",
    "created": "2025-10-31 14:23:45"
  }
]

Fix Broken Links

$ sym fix

The following broken links will be removed:

  ✗ oldtool → ~/bin/removed-tool

Remove 1 broken link(s)? (y/N): y
✓ Removed 1 broken link(s)!
