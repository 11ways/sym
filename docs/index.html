sym

A simple tool for managing symbolic links in ~/.local/bin

About Symbolic Links

A symbolic link (symlink) is a file that points to another file or directory. They're useful for making executables available in your PATH without copying files or modifying system directories.

The problem: managing symlinks manually can be tedious. You need to remember the exact ln -s syntax, keep track of what links to where, and manually check for broken links when files move.

That's where sym comes in. It's a small bash script that makes creating, listing, and maintaining symlinks straightforward. No magic, just a simpler interface for everyday symlink tasks.

Installation

# Install using Homebrew $ brew install 11ways/sym/sym # Verify installation $ sym --version # View documentation $ man sym
# One-line installation $ curl -fsSL https://raw.githubusercontent.com/11ways/sym/main/install.sh | bash # Or inspect first, then install $ curl -fsSL https://raw.githubusercontent.com/11ways/sym/main/install.sh -o install.sh $ less install.sh $ bash install.sh
# Clone and install $ git clone https://github.com/11ways/sym.git $ cd sym $ make install-local

Features

Simple Commands

Create, list, inspect, and remove links with straightforward commands.

Safety Checks

Confirmation prompts and dry-run mode prevent accidental changes.

Verify & Fix

Detect and remove broken links automatically.

Multiple Formats

Export to text, JSON, or CSV for scripting and automation.

Cross-Platform

Works on both macOS and Linux without modifications.

No Dependencies

Pure bash script with no external requirements.

Man Page Included

Complete documentation accessible via man sym command.

Usage Examples

Create a Link

$ sym sublime /Applications/Sublime\ Text.app/Contents/.../subl
Successfully created symbolic link:
  From: /Applications/Sublime Text.app/.../subl
  To:   ~/.local/bin/sublime

List Links

$ sym ls

Existing symbolic links in '~/.local/bin':

           sublime → /Applications/Sublime Text.app/.../subl
              code → /Applications/Visual Studio Code.app/.../code
            docker → /usr/local/bin/docker

Show Link Info

$ sym info sublime

  From:    ~/.local/bin/sublime
  To:      /Applications/Sublime Text.app/.../subl
  Created: 2025-10-31 14:23:45
  Size:    2.4MB

Verify Links

$ sym verify

Verifying symbolic links...

  ✓ sublime → /Applications/Sublime Text.app/.../subl
  ✓ code → /Applications/Visual Studio Code.app/.../code
  ✗ oldtool → ~/bin/removed-tool (broken)

Export to JSON

$ sym ls --format=json
[
  {
    "name": "sublime",
    "target": "/Applications/Sublime Text.app/.../subl",
    "status": "ok"
  }
]

Fix Broken Links

$ sym fix

The following broken links will be removed:

  ✗ oldtool → ~/bin/removed-tool

Remove 1 broken link(s)? (y/N): y
Removed 1 broken link(s)
